RA-L Supplementary Material¶

System Pipeline¶

The motion model computes average acceleration \(\bar{\mathbf{a}}\) and angular velocity \(\bar{\boldsymbol{\omega}}\) for the window of IMU measurements between two LiDAR scans. These control inputs are used to deskew the incoming LiDAR scan \(\mathcal{P}\) into the deskewed scan \(\mathcal{P}^*\).

The deskewed scan is first downsampled to \(\mathcal{P}^*_{merge}\) (which is later used to update the map). An optional second downsampling step produces \(\mathcal{Q}\); if this second step is disabled, \(\mathcal{Q} = \mathcal{P}^*_{merge}\).

For registration, \(\mathcal{Q}\) is matched against the VDB-based map by computing data associations via a nearest neighbor search with an association threshold. The motion model provides the initial guess for the first iteration of the optimization, while subsequent iterations use the current best estimate.

The iterative optimization minimizes a joint cost: an ICP cost from the associations, and an orientation regularization cost. The final estimated pose \(\mathbf{T}\) (after convergence) is used to transform \(\mathcal{P}^*_{merge}\) and update the map.

Trajectory Plots¶

In this section, we present trajectory plots using evo_ape (Absolute Pose Error). The trajectories are colored based on error magnitude, where “hotter” colors (reds/yellows) indicate higher error.

Oxford Spires (Radcliffe)¶

Shown above are two trajectory results when ablating the performance of the odometry system. The color scale represents the APE error. Please note that the range of error (color bar scale) is different in both plots; the full system trajectory has a lower total error than the no-regularization version. From this comparison, it can be seen that the full system (Ours) performs better on this sequence.

Car Dataset (Urban)¶

Similar to the Oxford Spires results, we once again see the full system performs (Ours) better than the ablated version (-AR). Additionally, when looking at the individual axes plots below, the specific benefits become clearer.

We can see that while the performance on X and Y across the full trajectory might be similar between ablations, there is noticeably less Z-axis drift in the full system version. Please note that “No adaptive regularization (blue)” corresponds to the -AR ablation and “Full system (green)” corresponds to the Ours ablation.

Car Dataset (Rural)¶

This behavior of reduced vertical drift is seen across other sequences as well. Below we examine the rural sequence.

Here we have zoomed into the start of the trajectory results on the 50 km long rural sequence data. We observe a similar behavior where the Z drift is reduced by enabling the adaptive regularization in the full system (green, corresponds to Ours ablation).

Here we plot the full XY trajectory results from all the methods which successfully ran on the rural sequence. The trajectories have been aligned (using Umeyama alignment) to the reference trajectory, which is defined in a GPS coordinate frame. Each approach experiences significant drift on this 50 km long sequence, and as a consequence of the alignment process, the trajectories appear well-aligned near the middle of the sequence but drift at the ends. Nevertheless, we see that our approach (red) is the one that remains closest to the reference trajectory overall.
